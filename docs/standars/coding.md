1. Una clase = una responsabilidad.

2. Si el código se reutiliza tres veces, pasa al Core.

3. No usar valores hardcoded.

4. Utilizar Enums cuando representen estados o tipos.

5. Todo comportamiento común debe vivir en Base o Traits.

6. Los Controllers no contienen lógica de negocio.

7. Los Services implementan la lógica de negocio.

8. Los Repositories acceden a los datos.

9. Las Actions representan un único caso de uso.

10. Todo código nuevo debe estar documentado.
